Try new technologies and architectures

I wonder if Elasticsearch would work better than Solr?

To test a new technology, start by making a new branch.

On the new branch, add the ElasticSearch service to your services.yaml file.

"Relationships" in the .platform.app.yaml file tell your app how to communicate with the new ElasticSearch service, and they will work on every new branch you make.

You may need build-time dependencies for your stack, such as Ruby gems, PIP packages, Composer libraries, or NPM stuff. You can specify that in YAML as well.

Deploy...

Deploy the new service by pushing your code to the new branch.

platform service:list, connect to Elastic...

You can connect to the new ElasticSearch service using the CLI tool to test it.

Share the new environment with others with the URL.

When you've tested and want to take your new technology live, just merge into Master and the application deploys with the new ElasticSearch service.

Since the infrastructure and its configuration is stored in Git alongside the code, you can roll back the infrastructure changes the same way you revert an unwanted change made to the code. Just git revert.
